Arquitetura

1. Módulo da geração de documentos
	
	Incumbido de criar e manipular os arquivos. Detém as funcionalidades de ler um determinado template no formato markdown, identificar suas chaves no formato padrão <<CHAVE>>, e em seguida, substituí-las pelos respectivos campos do dataset em consonância com o formato especificado abaixo:
		
	...
	
	Na sequência, gerará o documento em formato pdf e irá disponibilizá-lo no diretório padrão output.
	Este módulo opera com apenas um conjunto de dados, isto é, para um objeto, ele retorna apenas 1 documento.
	Este módulo tem como saída principal o método gera_documento().	
	

2. Módulo de tratamento de dados

	Responsável pela extração e filtragem do dataset a partir do path especificado pelo usuário. Aceita os seguintes formatos: csv, excel, html, blablabla.
1) Limpeza de linhas incompletas;
2) Remoção de duplicatas;
3) Conversão para o formato requerido no módulo da geração de documentos;

3. Módulo do envio de emails

	Administração do envio de emails pelo usuário por meio do endereço de email. Para tanto, o nome da coluna padrão deverá ser "email", caso contrário, o usuário deverá informar qual é a coluna designada para tal.
	Este módulo será invocado dentro do método gera_documentos(), realizando o envio do email, junto com o respectivo documento, para fins de assinatura ou emissão de declaração, por exemplo.
	Possui valor padrão FALSO, o que significa que o usuário deverá informar se deseja ou não enviar os arquivo para os emails.
	
4. Módulo gerador de documentos

	Este é o módulo final, chamado ao inserir o código CLI no terminal (main).
